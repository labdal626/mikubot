# 🚀 TUTORIAL HOSTING (Khusus HP)

Bot ini bisa di-host 24/7 GRATIS dari HP Android/iOS. Ikuti langkah berikut:

## 📋 Persiapan
**Yang dibutuhkan:**
- 📱 HP (Android/iOS)
- 📧 Email aktif
- 🌐 Browser (Chrome/Safari)
- ☕ Kopi (opsional 😉)

---

## Langkah 1 – Bikin Bot Discord & Ambil Token

1. Buka [discord.com/developers/applications](https://discord.com/developers/applications) di browser HP.
2. Login pakai akun Discord kamu.
3. Klik **New Application** (kanan atas), beri nama **MikuBot** lalu **Create**.
4. Di menu sebelah kiri, klik **Bot**.
5. Klik **Reset Token** → **Yes, do it!** → **Copy** token. **JANGAN SHARE TOKEN INI!** ⚠️
   - Simpan dulu di Notes/Notepad HP kamu.
6. Scroll ke bawah, di bagian **Privileged Gateway Intents**, AKTIFKAN ketiga toggle:
   - ✅ **PRESENCE INTENT**
   - ✅ **SERVER MEMBERS INTENT**
   - ✅ **MESSAGE CONTENT INTENT**
7. Klik **Save Changes**.

### Invite Bot ke Server
8. Di menu kiri, klik **OAuth2** → **URL Generator**.
9. Centang scopes: **bot** dan **applications.commands**.
10. Di bagian **Bot Permissions**, centang **Administrator** (paling mudah).
11. Copy URL paling bawah, buka di browser HP, pilih server kamu, klik **Authorize**.

---

## Langkah 2 – Upload Code ke GitHub

1. Buka [github.com](https://github.com), daftar/login.
2. Klik tombol **+** kanan atas → **New repository**.
3. Isi:
   - **Repository name**: `mikubot`
   - **Public** (biar gratis di Render)
4. Klik **Create repository**.
5. Di halaman repo yang baru dibuat, klik **uploading an existing file**.
6. **Extract** file ZIP `mikubot.zip` yang kamu download (pakai aplikasi seperti **ZArchiver** di Android atau **Files** di iOS).
7. Upload **semua file** dari folder hasil extract ke GitHub:
   - Drag-and-drop atau klik **choose your files**
   - **PENTING:** Upload SEMUA file termasuk folder `cogs/` dan `utils/`. Jangan zip lagi.
   - Cara mudah: pilih semua file (long-press file pertama lalu select all), lalu upload.
8. Scroll ke bawah, klik **Commit changes**.

---

## Langkah 3 – Deploy ke Render (Server Gratis)
Render adalah platform cloud gratis yang bisa deploy dari GitHub. Setiap perubahan di repo langsung auto‑deploy.

1. Buka [render.com](https://render.com) di browser HP, **Sign Up** pakai akun GitHub.
2. Setelah masuk, klik **New +** → **Web Service**.
3. Hubungkan akun GitHub-mu, pilih repository `mikubot`.
4. Atur pengaturan:
   - **Name**: `mikubot` (url akan `https://mikubot.onrender.com`)
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python main.py`
   - **Instance Type**: **Free** (pilih yang Free)
5. Scroll ke **Environment Variables**, tambahkan:
   - Key: `TOKEN`
   - Value: token bot Discord yang tadi disalin
6. Klik **Create Web Service**.

Render akan mulai build dan deploy (butuh beberapa menit). Setelah selesai, di dashboard akan muncul link URL, contoh `https://mikubot-xxxx.onrender.com`. Simpan URL ini.

Bot kamu sekarang online, tapi free tier Render akan **sleep setelah 15 menit tidak ada kunjungan**. Maka dari itu langkah terakhir.

---

## Langkah 4 – Biar Bot 24 Jam Nonstop pakai UptimeRobot
1. Buka [uptimerobot.com](https://uptimerobot.com), daftar/login.
2. Klik **Add New Monitor**:
   - **Monitor Type**: HTTP(s)
   - **Friendly Name**: MikuBot
   - **URL**: paste URL Render (misal `https://mikubot-xxxx.onrender.com`)
   - **Monitoring Interval**: 5 menit (gratis)
3. Simpan. UptimeRobot akan otomatis mengunjungi URL botmu setiap 5 menit, membuatnya tidak sempat tidur.

Sekarang **MikuBot tetap online 24/7, gratis, dan kamu punya kendali penuh!** 🎉

## ❓ Troubleshooting

### Bot tidak muncul online di Discord
- Pastikan TOKEN di Render benar (no spasi, no quote)
- Cek logs di Render → tab **Logs**
- Pastikan 3 Intents di Discord Developer aktif

### Slash command tidak muncul
- Tunggu 5-10 menit untuk Discord sync
- Atau kick + invite ulang bot dengan link OAuth yang baru

### Bot kena rate limit
- Render free tier punya batas, tunggu 1 jam atau upgrade

### Render error "no module named X"
- Cek `requirements.txt` ada di repo
- Trigger redeploy di dashboard Render

### "Application did not respond"
- Bot lagi sleep / lambat respon. Cek UptimeRobot aktif
